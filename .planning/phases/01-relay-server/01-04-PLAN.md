- `restartPolicyType: "ON_FAILURE"` with max 5 retries ensures the relay recovers from transient crashes
- No Dockerfile needed -- Nixpacks handles the build from package.json

This is the ONLY file change needed. The relay code itself requires NO modification.

The user must perform these steps manually. Claude cannot authenticate with Railway or create paid service projects.

**Step 1: Install Railway CLI (if not already installed)**
```bash
brew install railway
```
Or: `npm install -g @railway/cli`

**Step 2: Authenticate**
```bash
railway login
```
This opens a browser for OAuth. Complete the login flow.

**Step 3: Initialize Railway project in the relay directory**
```bash
cd relay
railway init
```
Select "Create new project" when prompted. Name it something like "keyauth-relay".

**Step 4: Configure the root directory**
Railway needs to know the service lives in `relay/`, not the repo root. In the Railway dashboard:
1. Go to the service Settings
2. Under "Source" or "Root Directory", set the root directory to `/relay`
Alternatively via CLI if supported: `railway settings --root-directory relay`

**Step 5: Set environment variables**
In the Railway dashboard (Settings > Variables) or via CLI:
```bash
railway variables set APNS_KEY_ID=<your-key-id>
railway variables set APNS_TEAM_ID=<your-team-id>
railway variables set APNS_KEY=<base64-encoded-p8-key>
railway variables set APNS_ENVIRONMENT=sandbox
```

To base64-encode your .p8 key file:
```bash
base64 -i AuthKey_XXXXXXXXXX.p8 | tr -d '\n'
```

Note: If you do not have APNs credentials yet, the relay will still deploy and work for WebSocket routing. APNs push will be disabled (the relay logs a warning and continues -- see index.ts line 18). You can add the APNs env vars later.

**Step 6: Deploy**
```bash
railway up
```
Or push to the linked GitHub repo branch to trigger auto-deploy.

**Step 7: Get the public URL**
```bash
railway domain
```
If no domain is assigned yet, generate one in Railway dashboard: Settings > Networking > Generate Domain. The URL will look like `keyauth-relay-production-XXXX.up.railway.app`.
