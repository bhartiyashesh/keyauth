<objective>
Scaffold the WXT Chrome extension project and build the foundational crypto, types, and storage modules that all subsequent plans depend on.

Purpose: Establish the extension project structure, install dependencies, and create the CryptoBox module that produces byte-identical output to iOS CryptoKit ChaChaPoly. This is the highest-risk custom code in the phase -- getting the wire format wrong breaks the entire end-to-end flow.

Output: A buildable WXT project with crypto.ts (X25519 + ChaCha20-Poly1305 + HKDF interop), types.ts (relay protocol types), and storage.ts (chrome.storage wrappers).
</objective>

<threat_model>
- **T1: Key material in storage** -- Private key from X25519 keypair must be discarded after deriving the shared key during pairing. Only the derived sharedKey is stored in chrome.storage.local. The privateKey is held temporarily in chrome.storage.session during the pairing window only.
- **T2: Weak randomness** -- Nonce generation uses `randomBytes(12)` from @noble/ciphers which delegates to `crypto.getRandomValues()`. Never use `Math.random()`.
- **T3: Wire format mismatch as interop vulnerability** -- If nonce is not correctly prepended/stripped, decryption will fail silently or produce garbage. Round-trip test validates this.
- **Mitigations applied:** Crypto module uses audited noble libraries (cure53 audit Sept 2024). HKDF parameters are hardcoded constants matching iOS exactly. Storage module never writes raw private keys to chrome.storage.local.
</threat_model>

    1. Create `extension/src/lib/crypto.ts` with these exact exports:

       ```typescript
       import { x25519 } from '@noble/curves/ed25519';
       import { chacha20poly1305 } from '@noble/ciphers/chacha';
       import { randomBytes } from '@noble/ciphers/utils';
       import { hkdf } from '@noble/hashes/hkdf';
       import { sha256 } from '@noble/hashes/sha2';

       const HKDF_SALT = new Uint8Array(0);           // matches CryptoKit Data()
       const HKDF_INFO = new TextEncoder().encode('KeyAuth-E2E');  // matches Data("KeyAuth-E2E".utf8)

       export function generateKeyPair(): { privateKey: Uint8Array; publicKey: Uint8Array } {
         const privateKey = x25519.utils.randomPrivateKey();
         const publicKey = x25519.getPublicKey(privateKey);
         return { privateKey, publicKey };
       }

       export function deriveSharedKey(privateKey: Uint8Array, peerPublicKey: Uint8Array): Uint8Array {
         const sharedSecret = x25519.getSharedSecret(privateKey, peerPublicKey);
         return hkdf(sha256, sharedSecret, HKDF_SALT, HKDF_INFO, 32);
       }

       export function seal(plaintext: Uint8Array, sharedKey: Uint8Array): Uint8Array {
         const nonce = randomBytes(12);
         const cipher = chacha20poly1305(sharedKey, nonce);
         const ciphertextWithTag = cipher.encrypt(plaintext);
         // Prepend nonce to match CryptoKit SealedBox.combined: nonce(12) || ciphertext || tag(16)
         const combined = new Uint8Array(12 + ciphertextWithTag.length);
         combined.set(nonce, 0);
         combined.set(ciphertextWithTag, 12);
         return combined;
       }

       export function open(combined: Uint8Array, sharedKey: Uint8Array): Uint8Array {
         const nonce = combined.slice(0, 12);
         const ciphertextWithTag = combined.slice(12);
         const cipher = chacha20poly1305(sharedKey, nonce);
         return cipher.decrypt(ciphertextWithTag);
       }

       export function uint8ArrayToBase64(bytes: Uint8Array): string {
         return btoa(String.fromCharCode(...bytes));
       }

       export function base64ToUint8Array(base64: string): Uint8Array {
         const binary = atob(base64);
         const bytes = new Uint8Array(binary.length);
         for (let i = 0; i < binary.length; i++) {
           bytes[i] = binary.charCodeAt(i);
         }
         return bytes;
       }
       ```
