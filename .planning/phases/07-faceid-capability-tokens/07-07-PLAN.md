<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Toggle flip → UserDefaults write | Already the Phase 6 Sync toggle's trust boundary — no new surface. |
| ContentView overlay → sheet covering | SwiftUI-level invariant: `.sheet` covers `.overlay` when presented. Benign here because toast is NEVER fired during the mint-flow sheet (Pitfall 3). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-05 | Elevation of Privilege | Toggle OFF does not instantly revoke in-flight window | accept (documented) | UI-SPEC §Interaction Patterns explicitly documents that the toggle does NOT force-revoke in-flight windows — the next revocation trigger (background, iCloud change, Timer expiry) handles it. Invisible to user. |
| T-7-08 | Spoofing / Repudiation | Toast not visible due to overlay-sheet interaction on first request | mitigate | Pitfall 3: sheet covers overlay while presented — benign because no toast ever fires during the mint flow. First request → sheet visible → mint on approve → sheet auto-dismisses → subsequent requests ARE silent and show toast above now-empty ContentView. FIDO-18 manual QA verifies the "subsequent" case on physical device. |
| T-7-CR1 | Spoofing | Copy drift in future refactors (e.g., "Enable trust window" replacing the verbatim label) | mitigate | Three new grep tests enforce the UI-SPEC verbatim strings at CI time. A refactor triggers a failing test unless the UI-SPEC is updated in the same commit. |
</threat_model>
